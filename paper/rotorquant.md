## Abstract

The rotation matrix $\Pi \in \mathbb{R}^{d\times d}$ is the computational bottleneck: for $d=128$ it requires 16,384 parameters and a dense matrix-vector multiply per vector. 

Our contribution. We replace $\Pi$ with Clifford rotors in $Cl(3,0)$, reducing parameters by 44x and with fused CUDA/Metal kernels achieving 10-31x speedup over TurboQuant while maintaining identical attention fidelity on real models.

We present RotorQuant, a reimagining of Google's TurboQuant [1] that replaces the $d\times d$ random orthogonal rotation matrix $\Pi$ with Clifford rotors $R=exp(B/2)$ in the geometric algebra $Cl(3,0)$. Instead of a matrix multiply $\Pi x$ requiring $d^{2}$ multiply-adds, RotorQuant performs the rotor sandwich product $RxR$ using only ~100 multiply-adds per vector exploiting the algebraic sparsity of rotors (4 of 8 multivector components are zero).

Fused GPU kernels (CUDA for NVIDIA, Metal for Apple Silicon) implementing the full pipeline achieve 10-19x speedup on NVIDIA and 9-31x speedup on Apple Silicon over TurboQuant's BLAS matmul, while using 44x fewer parameters (372 vs. 16,399 for $d= 128$).

Validated on real KV cache data from Qwen2.5-3B-Instruct, RotorQuant matches TurboQuant's attention fidelity (cosine similarity 0.990 vs. 0.991) and achieves higher top-1/top-5 retrieval accuracy at 4K context suggesting the Clifford rotor decorrelation better preserves directional structure of real attention heads.

## 1 Introduction

Large language models store key and value vectors for every token across every layer in the KV cache. At 8K tokens on Qwen2.5-3B (36 layers, 128-dim heads), this cache consumes 289 MB in FP16. On a 24 GB GPU, the KV cache not the model weights becomes the bottleneck for long context.

TurboQuant [1] compresses these vectors to 2-4 bits per coordinate via a two-stage process:

1. **Stage 1 (MSE):** Random orthogonal rotation $\Pi$ (via QR of a Gaussian matrix) decorrelates coordinates, enabling independent per-coordinate Lloyd-Max quantization.
2. **Stage 2 (QJL):** A 1-bit Quantized Johnson-Lindenstrauss [2] transform on the residual provides unbiased inner product estimation.

## 2 Background: Clifford Algebra $Cl(3,0)$

The geometric algebra $Cl(3,0)$ is generated by three orthonormal basis vectors $e_{1}$, $e_{2}$, $e_{3}$ with the relation $e_{i}e_{i}=+1$. A general element (multivector) has 8 components:

$$M = s + v_{1}e_{1} + v_{2}e_{2} + v_{3}e_{3} + b_{12}e_{12} + b_{13}e_{13} + b_{23}e_{23} + te_{123}$$

A rotor is an even-grade multivector of the form:

$$R = \cos(\theta/2) + \sin(\theta/2)\hat{B} \quad (2)$$

where $\hat{B}$ is a unit bivector specifying the rotation plane and $\theta$ is the rotation angle. R has only 4 non-zero components: $[s, 0, 0, 0, b_{12}, b_{13}, b_{23}, 0]$, normalized so $R\tilde{R}=1$.

The sandwich product $Rv\tilde{R}$ rotates a vector $v$ while preserving all algebraic structure (norms, inner products, outer products, and grades).

## 3 Method: RotorQuant

### 3.1 Vector Chunking and Rotor Decorrelation

Instead of one $d\times d$ matrix, RotorQuant chunks the $d$-dimensional vector into groups of 3 dimensions and rotates each group with its own Clifford rotor:

1. **Embed:** Reshape $x\in\mathbb{R}^{d}$ into $\lceil d/3\rceil$ groups of 3 dimensions, each embedded as a grade-1 multivector $[0, x_{1}, x_{2}, x_{3}, 0, 0, 0, 0]$.
2. **Rotate:** Apply per-group rotor sandwich $R_{g}v_{g}\tilde{R}_{g}$ for each group $g$.
3. **Quantize:** Grade-aware Lloyd-Max quantization on the rotated multivector (different codebooks for scalar, vector, bivector, and trivector grades).
4. **Un-rotate:** Apply inverse sandwich $\tilde{R}_{g}q_{g}R_{g}$
5. **Extract:** Reshape back to $\mathbb{R}^{d}$.

### 3.2 Rotor Sparsity Exploitation

Since rotors have only 4 non-zero components, the geometric product $RM$ reduces from 64 to 28 FMAs:

$$r_{0}=sx_{0}-b_{12}x_{4}-b_{13}x_{5}-b_{23}x_{6}$$
$$r_{1}=sx_{1}+b_{12}x_{2}+b_{13}x_{3}+b_{23}x_{7}$$
$$r_{2}=sx_{2}-b_{12}x_{1}+b_{23}x_{3}-b_{13}x_{7}$$
$$r_{3}=sx_{3}-b_{13}x_{1}-b_{23}x_{2}+b_{12}x_{7} \quad (3)$$
$$r_{4}=sx_{4}+b_{12}x_{0}$$
$$r_{5}=sx_{5}+b_{13}x_{0}$$
$$r_{6}=sx_{6}+b_{23}x_{0}$$
$$r_{7}=sx_{7}-b_{23}x_{1}+b_{13}x_{2}-b_{12}x_{3}$$

The full sandwich requires two such products (forward + reverse), totaling ~56 FMAs per group, or ~2,400 FMAs for $d=128$ (43 groups) compared to TurboQuant's 16,384 FMAs.

### 3.4 QJL Residual Correction

Stage 2 is identical to TurboQuant: the quantization residual $r=x-\hat{x}$ is projected through a random Gaussian matrix $S$ and only the signs are stored (1 bit per dimension).

The unbiased inner product estimator is:

$$ \frac{\pi}{2} (Sy, \text{sign}(Sr)) \quad (4)$$

## 4 Fused GPU Kernels

The entire pipeline (embed $\rightarrow$ rotor sandwich $\rightarrow$ quantize $\rightarrow$ inverse $\rightarrow$ extract) is implemented as a single GPU kernel launch on both platforms:

* **CUDA (`rotor_fused_kernel.cu`):** Each thread handles one (batch, group) pair. Rotors and centroids are loaded into shared memory. Supports float16, float32, and bfloat16.
* **Metal (`rotor_fused.metal`):** Same algorithm via Metal compute shader. Rotors and centroids in threadgroup memory. Compiled to metallib via xcrun.

Why fused kernels win: TurboQuant's $\Pi x$ is a single cuBLAS/Accelerate GEMM call highly optimized but fundamentally $O(d^{2})$. RotorQuant's fused kernel does $O(d)$ FMAs with all data staying in thread-local registers, eliminating memory round-trips between pipeline stages.

### 5.3 MSE Distortion ($d=128$, 2000 unit vectors)

$$(y, x) \approx (y, X_{mse}) + \frac{\pi}{2} ...$$

TurboQuant achieves lower MSE because its full $d\times d$ rotation exactly induces the Beta distribution that Lloyd-Max was optimized for. RotorQuant's block-diagonal rotation (groups of 3) changes the per-component distribution. However, the QJL residual correction compensates, and on real model data the accuracy gap disappears (Section 5.6).

## 7 Discussion

**Why RotorQuant works despite higher synthetic MSE.** The QJL residual correction (Stage 2) dominates inner product accuracy it compensates for a weaker Stage 1. Real KV cache vectors are not random unit vectors; they live on low-rank manifolds where the rotor's structure-preserving properties help.

**When to use RotorQuant over TurboQuant.**
* **Maximum throughput:** Fused kernel is 10-31x faster.
* **Parameter-constrained settings:** 44-3,063x fewer params.
* **Apple Silicon:** Metal shader with no CUDA dependency.
* **Geometric data:** Rotor sandwich preserves full algebraic structure.

**Limitations.**
* The full $Cl(3,0)$ geometric product table has a sign error in some terms (identified by associativity test failure). RotorQuant only uses the sparse rotor path, which is correct.
* Block-diagonal rotation (groups of 3) does not decorrelate between groups, potentially limiting MSE on some distributions.
* Grade-aware quantization adds codebook complexity vs. TurboQuant's single codebook.

## 8 Related Work

**KV cache compression.** TurboQuant [1] and PolarQuant [3] use random rotation for coordinate decorrelation. QJL [2] provides 1-bit unbiased inner product quantization. KIVI [5] and KVQuant [6] use per-channel quantization with calibration data.

**Geometric algebra in ML.** CliffordNet [4] builds pure-GA vision backbones with $O(N)$ complexity. Clifford Algebras have been used for equivariant neural networks [7], molecular dynamics [8], and robotics [9].

## 9 Conclusion

RotorQuant demonstrates that Clifford rotors are a practical replacement for random orthogonal matrices in vector quantization. The combination of 44x fewer parameters, 10-31x faster fused kernels, and matching real-model attention fidelity makes it a compelling alternative to TurboQuant for KV cache compression.
